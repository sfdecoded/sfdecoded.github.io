Named Credential Authentication Error

A callout using a Named Credential received a 401 Unauthorized or 403 Forbidden response, indicating the credential's stored token is expired, the OAuth flow was not completed, or the external system's API key or certificate has changed.

error
System.CalloutException: Unable to complete callout — Named Credential authentication failed. HTTP/1.1 401 Unauthorized.

What does this error mean?

Named Credentials store authentication details (OAuth tokens, API keys, certificates, basic auth) for external endpoints and inject them into HttpRequest headers automatically. When the stored credential is invalid — because an OAuth token has expired and failed to refresh, a password was rotated, or an API key was revoked — the external server rejects the request with a 4xx status and Salesforce surfaces it as a callout exception.

Common Causes

1. OAuth token expired and refresh failed

OAuth 2.0 access tokens are short-lived. Named Credentials with OAuth auth protocols automatically attempt token refresh, but if the refresh token itself has expired (or was revoked by the external IdP), the credential becomes permanently invalid until re-authenticated manually.

2. Password or API key rotated externally without updating the Named Credential

The external system rotated its API key or the service account password was changed in the target system, but the Named Credential in Salesforce Setup still holds the old value. Every callout returns 401 until the credential is updated.

3. Named Credential was never authenticated (OAuth flow not completed)

A Named Credential using the Per User principal type requires each user to individually complete the OAuth flow via the Authentication Settings for External Systems link on their user record. If a user hasn't completed the flow, their callouts return 401.

How to Fix It

Solution 1: Always use Named Credentials — never hardcode auth

Use the callout:CredentialName endpoint prefix. Named Credentials are the only secure way to manage external auth in Salesforce — they encrypt credentials at rest, handle token refresh automatically, and are version-controlled in metadata.

Apex — Bad Example
// ❌ BAD: hardcoded credentials — brittle and insecure
HttpRequest req = new HttpRequest();
req.setEndpoint('https://api.example.com/data');
req.setHeader('Authorization', 'Bearer sk-live-abc123secret');
req.setMethod('GET');
HttpResponse res = new Http().send(req);
Apex — Correct: Named Credential
// ✅ GOOD: Named Credential handles auth transparently
HttpRequest req = new HttpRequest();
req.setEndpoint('callout:My_External_API/data');
req.setMethod('GET');
// No Authorization header needed — Named Credential injects it
HttpResponse res = new Http().send(req);

if (res.getStatusCode() == 401) {
    // Token may have expired — Named Credential auto-refreshes
    // on next request; log and allow retry logic to handle
    System.debug('Auth error — token refresh in progress');
}

Solution 2: Diagnose 401/403 responses with detailed logging

When a callout fails, log the full response status, WWW-Authenticate header, and body to help distinguish between an expired token (401), missing scope (403), and a misconfigured endpoint URL (404).

Apex — Diagnosing Auth Issues
// Check response status and body for auth error details
HttpResponse res = new Http().send(req);

System.debug('Status: '   + res.getStatusCode());
System.debug('Headers: '  + res.getHeader('WWW-Authenticate'));
System.debug('Body: '     + res.getBody());

if (res.getStatusCode() == 401) {
    throw new CalloutException(
        'Authentication failed for Named Credential. ' +
        'Check credential setup in Setup > Named Credentials. ' +
        'Response: ' + res.getBody()
    );
} else if (res.getStatusCode() == 403) {
    throw new CalloutException(
        'Authorisation denied — credential lacks required scopes.'
    );
}

Solution 3: Re-authenticate the Named Credential

For OAuth Named Credentials, navigate to Setup → Named Credentials, open the credential, and click Edit → Save to trigger re-authentication. For Per User credentials, affected users must visit My Settings → Authentication Settings for External Systems and re-authorise. For Password or API Key credentials, update the stored value to match the rotated secret.

💡
Pro Tip: For OAuth Named Credentials, set up a Salesforce Health Check dashboard or a scheduled Apex job that makes a lightweight GET /ping or /health call against each Named Credential every hour. A 401 response triggers an alert email to your integration team — catching token expiry before business-critical processes fail at peak hours.
